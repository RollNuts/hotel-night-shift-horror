# CODEX WORKING RULES

Status: active instructions for Codex work in this repository.

## First Rule

This is a first-person, single-location, night-shift business hotel horror game. Do not re-open genre selection.

## Codex Decision Frame

- Build: the smallest change that deepens hotel work, fear, production quality, or release safety.
- Do not build: inherited Fourfold/Unity ideas, online systems, open-world scale, combat-first systems, or code-only completion.
- Market reason: every action must strengthen the first Steam screenshot or trailer segment.
- Unreal reason: respect Blueprint-first workflow, binary asset ownership, Lumen/audio/camera evidence, and packaging constraints.
- Solo-dev/low-cost reason: reduce scope, avoid paid/recurring dependencies, and prefer original/simple assets when safe free assets do not fit.
- Art/audio bar: every gameplay task includes visual, camera, SFX, ambience, UI, level, performance, and capture readiness.
- No-small-room bar: test harnesses are allowed but cannot count as product progress.
- Steam quality bar: report public-readiness, evidence, placeholders, licenses, and performance.
- Veripsa unit: work must fit a Veripsa task with scope, non-goals, risks, and evidence.

## Before Work

- Read `REQUIRED.md`.
- Confirm the core action affected.
- Confirm the hotel area affected.
- Confirm no secret, paid asset, or unclear-license asset is being introduced.
- Check placeholder and asset ledger requirements.
- Check whether a binary Unreal asset needs ownership.
- Define evidence before implementation.
- For PR work, trigger Veripsa Core through the GitHub App by marking the PR
  ready for review when a Core coordination signal is needed.
- Read the Veripsa GitHub check/comment for traffic state: reservations,
  unknown/unindexed paths, possible collisions, and land-order risk. Local
  `Veripsa/` files are task records, not the Core traffic signal.

## Safe Tool And Asset Rule

Already installed tools and free official tools may be used. Free assets, music, SFX, and 3D models may be used only after license review.

Do not:

- Spend money.
- Start subscriptions.
- Expose tokens or credentials.
- Bypass software, platform, or asset terms.
- Commit raw restricted assets.
- Use unclear-license material.

## Completion Rule

Never report only "implemented" or "works."

Report:

- Player experience improvement.
- Hotel-level visual result.
- Audio/ambience result.
- What became scarier or clearer.
- Screenshot/video/log evidence.
- Performance impact.
- Remaining placeholders.
- License status.
- Next quality risk.
